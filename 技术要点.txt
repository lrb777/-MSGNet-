================================================================
  MSGNet 多尺度图学习因子挖掘系统 — 技术要点
================================================================

【项目一句话介绍】
基于 MSGNet 多尺度图神经网络，从行业ETF价格时序中自动挖掘
传统方法难以发现的「关系类因子」，并接入完整的多因子选股
验证框架（IC检验 → 因子合成 → Walk-Forward 回测）。

----------------------------------------------------------------
一、研究背景与核心思路
----------------------------------------------------------------

传统量化因子（动量、价值、质量）主要刻画单个标的自身属性，
难以捕捉标的之间的动态关联结构。

本项目的核心思路：
  - 将 MSGNet 作为「特征提取器」而非预测模型
  - 模型学到的邻接矩阵（股票间图关系）本身就是因子
  - 从图结构中提取传统因子无法表达的「网络中心性」信号

----------------------------------------------------------------
二、完整研究流程（7个阶段）
----------------------------------------------------------------

阶段一  数据准备
  - 标的池：20只行业ETF（覆盖宽基/金融/科技/消费/医疗/能源等）
  - 数据源：adata.fund.market.get_market_etf（解决海外访问限制）
  - ETF无前复权参数问题：用 change_pct 链式重建前复权价格
    adj_t = adj_{t-1} × (1 + change_pct_t / 100)
  - 自动检测 EFFECTIVE_START：取所有ETF共同有数据的最早日期，
    避免硬编码导致的数据对齐问题
  - 时序划分：6:2:2（训练/验证/测试），严格按时间顺序，不随机打乱

阶段二  预处理
  - 对数收益率：解决价格非平稳性
  - 截面 z-score：消除市场整体涨跌对截面信号的干扰
  - ±3σ 极端值裁剪：防止黑天鹅日破坏标准化结构

阶段三  模型训练（MSGNet）
  - 多尺度图学习：top_k=3 个 FFT 尺度，每层学习不同周期下的图结构
  - 关键超参约束：GraphBlock 的 Conv2d kernel = (d_model - N + 1, 1)，
    要求 d_model ≥ N（节点数）
    → ETF20：d_model=64，kernel=(45,1)，感受野充裕
    → CSI300：d_model=296，kernel=(4,1)，感受野极小（已知缺陷）
  - 训练目标：MSE 损失预训练，让模型学到有意义的邻接矩阵

阶段四  因子提取（三类因子）
  ① 图结构因子（核心）
     - 入度（in_degree）：被其他ETF影响的程度，衡量「跟随性」
     - PageRank：在整个关联网络中的重要程度
     - 尺度差异（scale_diff）：同一层内长短周期图结构的差异，
       衡量该ETF是否处于趋势转折点
     - 注：出度因softmax行归一化恒为1，已排除

  ② 隐层 Embedding 因子
     - 提取最后一层隐层表示 [N, d_model]
     - PCA 降维至10维正交因子，保留主要信息

  ③ 周期特征因子
     - FFT 识别主周期、振幅、周期稳定性
     - amplitude：当前周期振幅强度，高振幅 = 高波动 = 高Beta

阶段五  因子验证
  - IC 检验：截面 Spearman 秩相关，验证预测力
    通过门槛：|IC均值| > 0.03 且 |IC_IR| > 0.5
  - 分组回测：按因子值分5组，验证收益单调性
  - IC衰减分析：验证 h=1/5/10/20 日的预测力衰减曲线
  - 因子中性化：剥离市值/Beta暴露，提取纯Alpha信号

阶段六  因子合成与回测
  - IC加权合成：各因子按历史 IC 均值加权
  - Walk-Forward 回测：滚动窗口，测试集不重叠，防止未来函数
  - 交易成本：单边万分之5（佣金+滑点），ETF无印花税

阶段七  滚动更新机制
  - 月度滚动训练：用最近2年数据重新训练，保持因子时效性

三、关键工程难点与解决方案
----------------------------------------------------------------

问题1：海外访问国内数据源被封
  解决：adata.fund.market.get_market_etf（同花顺百度源，海外可访问）
  排查过程：逐一测试 akshare/eastmoney/sina/qq/east，全部返回空

问题2：ETF无前复权参数
  解决：change_pct 链式重建，增量更新时以缓存最后 adj_close 为锚点，
  保持价格序列连续性

问题3：硬编码起始日期导致训练数据含虚假零收益率
  根因：部分ETF上市晚于 START_DATE，早期行为 NaN，ffill() 补填后
        该段价格不变，计算出的对数收益率为0。这些假零值进入截面
        z-score，系统性地压低该ETF的标准化值，污染图结构学习。
  解决：自动检测 EFFECTIVE_START = max(所有ETF的 first_valid_index)
        从该日期起所有ETF均有真实数据，彻底消除填充值引入的偏差。

问题4：CSI300 模型图结构学习失效
  根因：MSGNet 的 GraphBlock 内部有一个卷积层：
        Conv2d(kernel_size=(d_model - N + 1, 1))
        kernel 高度决定了特征维度上的感受野大小。
        CSI300（N=293）要求 d_model≥293，取296时 kernel=(4,1)，
        退化为近点卷积，卷积层无法提取节点间的结构特征，
        邻接矩阵的梯度极弱，训练后图结构接近随机初始化。
        ETF20（N=20）取 d_model=64，kernel=(45,1)，
        感受野覆盖特征空间的70%，能有效学习节点差异。
  解决：先用 ETF20 验证图学习有效性，CSI300 需另行设计降维方案。

问题4：因子时间序列缺失（静态截面 vs 动态序列）
  根因：只在训练结束后提取一次邻接矩阵快照
  解决方向：滚动窗口推断，逐日生成 [T, N] 的因子矩阵

问题5：多空回测反映Beta而非Alpha
  根因：amplitude 是高Beta因子，2022-2026牛市中空头亏损
  解决：因子中性化（回归剥离市值/Beta暴露）

问题6：Windows DataLoader 多进程死锁
  解决：num_workers=0

四、因子中性化判断标准
----------------------------------------------------------------

中性化的目的：剥离因子中混入的系统性风险暴露（Beta、行业、市值），
保留因子真正的 Alpha 部分。不是所有因子都需要中性化。

【需要中性化的信号】

信号1：正向比例极端（< 30% 或 > 70%）
  正常 Alpha 因子的正向比例通常在 55~70% 之间，在不同市场环境下
  方向会有变化。如果正向比例只有 8%，说明因子在几乎所有时期都指向
  同一方向——这种极端一致性往往是风险因子在背后驱动，而非真实预测力。

信号2：IC 随预测期单调增大且不衰减
  真正的 Alpha 信号会随时间衰减（h=1 > h=5 > h=10 > h=20）。
  如果 IC 从 h=1 到 h=20 持续增大，说明因子在捕捉长期趋势性暴露
  （如市场 Beta、行业偏向），而非短期可利用的定价偏差。

信号3：回测中多头或空头与市场高度同向
  如本项目 amplitude：做空高振幅组，该组净值却涨到 2.2（市场仅 1.15），
  说明因子在做方向性的市场赌注，而不是选出了真正的 Alpha 标的。

【常见需要剥离的暴露】
  - 市场 Beta：因子值高的标的普遍是高 Beta 品种（振幅因子典型）
  - 行业偏向：因子值集中在某几个行业
  - 市值因子：A股中大量因子与市值高度相关

【不需要中性化的信号】
  正向比例在 55~80% 之间、IC 衰减正常、多空组合独立于市场方向，
  说明因子本身方向稳定，强行中性化反而可能破坏有效信息。
  本项目中 period_stability（正向比例 80.5%）和 scale_diff_L1
  （正向比例 74.8%）属于此类，无需中性化。

【中性化操作】
  本质是截面回归取残差：
    factor_neutralized = factor - β × risk_exposure
  用风险因子（如市场收益率、行业哑变量）对原始因子做横截面回归，
  取残差。残差剔除了风险暴露，再验证残差 IC 是否仍然显著。

六、因子质量评价体系（IC检验）
----------------------------------------------------------------

IC 检验是量化因子研究的核心评价工具，通过以下四个维度判断因子是否有效：

【IC均值】预测力强弱
  IC = 截面 Spearman 秩相关系数（因子值 vs 未来N日收益率）
  每个交易日算一个截面IC，取时间序列均值。
  |IC均值| > 0.03  → 有一定预测力（最低门槛）
  |IC均值| > 0.05  → 较好
  |IC均值| > 0.10  → 强因子
  负值 = 反向因子（因子值高→未来收益低），取负号使用，不影响有效性。

【IC_IR】信号稳定性（最重要）
  IC_IR = IC均值 / IC标准差，类似因子层面的夏普比率。
  衡量因子在不同时期预测方向是否一致。
  |IC_IR| > 0.5  → 信号稳定（门槛）
  |IC_IR| > 1.0  → 非常稳定，可信度高
  IC均值高但IC_IR低 → 因子时灵时不灵，实盘危险，不可用。

【正向比例】方向一致性
  每个截面上 IC > 0 的天数占比。
  正向因子应 > 55%；反向因子应 < 45%。
  极端值（如正向比例仅8%）说明反向信号极强且稳定。

【IC衰减曲线】调仓频率参考
  对比 h=1/5/10/20 日的IC值变化趋势：
  IC在h=5最高  → 因子适合周频调仓（每周换仓）
  IC随h单调增大 → 因子适合月频调仓（每月换仓）
  本项目ETF20结果：多数因子在h=10~20最强，适合月频调仓。

【本项目ETF20验证结果（etf20_1，未中性化）】
  通过门槛共8个因子，关键结论：
  - amplitude（|IC_IR|=1.405）：最强反向因子，疑含Beta暴露
  - period_stability（IC_IR=+0.897）：正向因子，正向比例80.5%，
    可能是真Alpha信号（与Beta暴露无关）
  - scale_diff_L1（IC_IR=+0.676）：正向因子，正向比例74.8%
  - in_degree_L2_S3、pagerank_L2_S3：反向因子，深层图结构信号

七、滚动更新机制
----------------------------------------------------------------

【为什么需要滚动更新】

模型训练完后，学到的邻接矩阵和因子值固定在训练集那段时间的市场
结构上。随着时间推移，股票间关系会变化（如某行业板块从高相关变
为分化），静态因子的预测力会逐渐衰减，最终失效。

滚动更新定期用最新数据重训模型、重提因子，保持因子时效性。

【滚动更新 vs 静态因子 Walk-Forward 的区别】

  静态因子 Walk-Forward（本项目当前状态）：
    训练一次 → 提取因子一次 → 把同一个因子拆成多个窗口评估
    → 本质是"市场环境压力测试"，不是真正的样本外验证

  滚动更新 Walk-Forward（升级后）：
    每个测试窗口前重新训练 → 每段时间用对应时间训练的因子
    → 才是真正的样本外泛化测试，才能生成因子时间序列

【滚动更新的三个核心价值】

  1. 因子时效性：模型始终反映最新市场结构
  2. 真正样本外验证：测试窗口的因子从未见过测试期的数据
  3. 实盘可落地：真实量化策略必须有更新机制

【实现方式（滚动窗口）】

  参数：训练窗口500日（约2年），测试窗口63日（约3个月），步长63日
  流程：
    Fold 1：train[0:500]   → 训练模型 → 提取因子 → test[500:563]
    Fold 2：train[63:563]  → 训练模型 → 提取因子 → test[563:626]
    Fold 3：train[126:626] → 训练模型 → 提取因子 → test[626:689]
    ...
  每个 Fold 独立训练，因子仅基于该 Fold 训练集内的数据。

【与因子时间序列的关系】

  滚动更新是生成"因子时间序列"的方式：
  每个 Fold 产生一批因子值 → 拼接所有测试窗口的因子 → [T, N] 矩阵
  这才是完整的逐日动态因子，能做逐日 Beta 中性化和真正的 IC 检验。

八、技术栈
----------------------------------------------------------------

深度学习：  PyTorch, MSGNet（多尺度图卷积+Transformer）
图学习：    动态图邻接矩阵, PageRank, FFT多尺度分解
量化框架：  自研 IC检验/分组分析/Walk-Forward回测框架
数据工程：  adata, pandas, numpy, 增量缓存, 多线程并发下载
可视化：    matplotlib（IC分布/分组收益/因子相关性热图）
环境：      Python 3.x, conda, Windows 11, CUDA GPU
