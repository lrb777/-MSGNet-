## 阶段四补充：单因子收益显著性检验（Beta控制版）

已新增 `src/factor_return_test.py`，把截图中的“单因子显著性检验”改造为适合 ETF20 的版本：

1. 不做行业中性化：ETF本身就是行业/主题标的，行业中性化会剥离研究对象。
2. 只控制历史 Beta：用训练集收益计算每只ETF相对等权市场的Beta。
3. 因子先对Beta取残差，再做每日截面回归：`未来d日收益 = 常数项 + Beta暴露 + 因子残差暴露 + 残差`。
4. 对 d=1/5/10/20 重复检验，输出年化因子收益、IR、普通t值、Newey-West t值；显著门槛为 `|NW_t值| >= 2 且 |IR| >= 0.5`。

输出文件：`facts/runs/<run_id>/validation/factor_return_report.csv`

### 新增检验的主要结果

| 因子 | 最佳预测期 | 年化因子收益 | 因子IR | NW_t值 | 方向 |
|------|-----------|--------------|--------|--------|------|
| dominant_period | h=20 | -4.20% | -5.635 | -3.635 | 反向 |
| amplitude | h=20 | +3.31% | +4.536 | +2.838 | 正向 |
| pagerank_L1_S2 | h=20 | +4.04% | +4.404 | +2.799 | 正向 |
| in_degree_L1_S1 | h=20 | -3.69% | -4.255 | -2.652 | 反向 |
| in_degree_L1_S2 | h=20 | +4.01% | +3.965 | +2.552 | 正向 |
| scale_diff_L2 | h=20 | -3.45% | -3.865 | -2.381 | 反向 |

### 解释

- 新增检验确认：MSGNet图结构因子和周期因子在控制静态Beta后，仍能产生显著横截面因子收益。
- 该检验是对IC检验的增强，但不能直接替代组合回测；ETF20截面小，单因子回归方向对样本和口径敏感。
- `period_stability` 在原IC口径下表现较好，但在Beta控制的单因子收益口径下未通过显著性门槛（h=20：IR=1.137，NW_t=0.703），后续应优先作为组合辅助因子，而不是单独交易信号。

## 阶段五：回测结果

### 原始回测（未中性化，amplitude + period_stability IC加权）
- 全期：年化约 -30%，Sharpe 约 -1.5
- 问题：空头（高振幅ETF）在牛市中持续上涨，多空亏损

### Beta中性化后回测（amplitude截面回归取残差）

| 指标 | 全期 | 测试集（样本外） | 多头单独（测试集） |
|------|------|----------------|-----------------|
| 年化收益 | -14.63% | -17.05% | **+26.39%** |
| Sharpe | -1.512 | -2.102 | **1.623** |
| 最大回撤 | -58.43% | -20.15% | -9.47% |
| 日胜率 | 47.20% | 45.91% | 55.64% |

### 关键结论
- **多头 Sharpe=1.623，超过研究目标（>1.0）** ✅
- 低振幅+高稳定性 ETF 选股是真实有效的 Alpha 信号
- 空头仍亏损：静态 Beta 中性化精度有限，高振幅 ETF 含有未被剥离的暴露
- 中性化有效：全期年化从约-30%改善至-14.63%，方向正确但不彻底

### 阶段判断
- ETF20 多空结构不作为后续主线，空头腿问题主要是结构性 Beta 暴露。
- 多头组合保留为后续 ETF 选择/增强配置策略的核心载体。
- 指数/市场对冲仅作为 Beta 归因验证工具，用于判断多头收益中 Alpha 与市场 Beta 的比例，不作为默认最终策略方向。

## 阶段六：静态 Walk-Forward 结果（walk_forward.py）

- 窗口长度：63交易日（约3个月），共20个窗口
- 正收益窗口：2 / 20（10%）
- 结论：多头Low-Beta ETF集中，空头High-Beta ETF集中，纯粹Beta方向押注，牛市必亏

## 阶段七：滚动更新 Walk-Forward 结果（rolling_update.py）

| 指标 | 结果 |
|------|------|
| Fold数量 | 12 |
| 正收益Fold | 2 / 12（17%） |
| 平均年化收益 | -12.04% |
| 平均Sharpe | -1.189 |
| Sharpe标准差 | 1.223 |

### 关键结论
- 滚动重训练与静态因子结果一致，均约2/12正收益
- 根本原因：ETF20标的数N=20，TOP_PCT=20%只能选4只多头+4只空头；空头恒为高振幅高Beta ETF，牛市系统性上涨
- Beta中性化（N=20截面OLS，自由度极低）无法有效剥离暴露
- **结论：多空结构在ETF20上不可行，空头问题是结构性的，而非优化问题**
- **核心Alpha信号已验证：低振幅+高周期稳定性选股有效（多头测试集Sharpe=1.623）**

## 关键技术约束

| 约束 | 原因 |
|------|------|
| d_model ≥ N_ETF | GraphBlock Conv2d kernel=(d_model-c_out+1,1) 要求 |
| ETF20: d_model=64, batch_size=32 | kernel=(45,1)，感受野充裕；显存压力小 |
| num_workers=0 | Windows DataLoader多进程坑 |
| get_market_etf | 海外访问，其余接口封锁 |
| 静态因子中性化精度有限 | 截面一次性回归 vs 逐日动态回归 |

## 下一阶段优化路线

优先级建议：

1. **ETF 多头选择/增强配置策略构建**：保留已经验证有效的低振幅+高周期稳定性多头信号，重点优化 TOP_PCT、换仓频率、持仓集中度、交易成本和组合稳定性。
2. **Beta 归因验证**：使用 ETF20等权市场、沪深300或中证800等基准对多头组合做固定比例和滚动 Beta 对冲，目的不是确定最终策略必须加对冲，而是判断多头收益中 Alpha 与市场 Beta 的比例。
3. **ETF20 逐日动态 Beta 中性化对照实验**：作为补充验证，评估静态 Beta 中性化是否低估了空头风险；但由于 ETF20 截面仍小，不作为主线优化。
4. **扩展到 CSI300 个股池**：作为第二阶段研究方向。CSI300 截面更适合多空和风险中性化，但会引入更高训练压力、个股噪声、停牌涨跌停、成分股变动和幸存者偏差处理成本。

下一步建议新增：

```text
src/hedged_long_backtest.py
```

已完成第一版 Beta 归因验证实现，默认使用 ETF20 等权市场作为对冲基准，并输出全期/测试集绩效、对冲 Beta、收益序列和可视化。

Beta 归因评估内容：

- 多头组合原始收益
- 指数/市场对冲后收益
- 对冲前后 Beta
- Sharpe、最大回撤、Calmar、日胜率
- 不同对冲工具：沪深300、中证800或ETF20等权市场
- 不同对冲比例：固定比例、滚动 Beta 比例

### 1. 标准主流程

```powershell
python .\src\data_pipeline.py
python .\src\preprocess.py
python .\src\train.py
python .\src\factor_extract.py
python .\src\factor_validate.py
python .\src\factor_return_test.py
```

运行顺序说明：

1. `data_pipeline.py`：下载/增量更新 ETF20 行情，生成 `data/etf20/price.csv`。
2. `preprocess.py`：生成 MSGNet 输入 `returns.csv` 和 `returns.npy`。
3. `train.py`：训练 MSGNet，生成 checkpoint。
4. `factor_extract.py`：从 checkpoint 提取图结构因子、Embedding 因子、周期因子。
5. `factor_validate.py`：跑 IC 检验和分组分析。
6. `factor_return_test.py`：跑 Beta 控制单因子收益显著性检验。
7. `factor_neutralize.py`：对 `amplitude` 做 Beta 中性化。
8. `backtest.py`：跑静态多空/多头回测。
9. `walk_forward.py`：跑静态因子的 Walk-Forward 稳健性检验。
10. `hedged_long_backtest.py`：跑 ETF 多头组合 + ETF20等权市场对冲回测。
